# 実装タスク — DSN 指定による erdm ファイル生成機能

設計（`design.md`）と発見ログ（`research.md`）、および直前のレビュー指摘（Title 既定値推定の責務集約 / `mysql://` プレフィックスと go-sql-driver/mysql DSN の関係明確化 / CLI 配線の責務分散）を実装に落とし込んだタスクリスト。各タスクは段階的に積み上げる構成で、並列実行可能なものに `(P)` マーカーを付ける。

- [ ] 1. プロジェクト基盤整備
- [x] 1.1 DB ドライバ依存の追加
  - PostgreSQL ／ MySQL ／ SQLite の 3 ドライバを純 Go 実装で追加し、CGO 不要の構成とクロスコンパイル成立を維持する
  - 追加直後に既存ビルドおよび既存テストが回ることを確認する
  - _Requirements: 12.3_
- [x] 1.2 (P) イントロスペクション専用パッケージのスケルトン作成
  - DB 接続・スキーマ取得・モデル変換を担う独立パッケージを新設し、外部公開はドメイン操作 1 関数に限定する設計とする
  - パッケージ内のサブモジュール分割（ドライバ別アダプタ／共通変換／値オブジェクト／ドライバ推定／DSN マスク／タイトル解決）の枠組みを用意する
  - _Requirements: 12.1, 12.2_

- [ ] 2. CLI サブコマンドの分岐と引数解析
- [x] 2.1 import サブコマンドの分岐配線
  - 第 1 引数が import を示す場合に既存サブコマンドのフラグ解析へ進まず、専用ハンドラへ委譲する
  - 既存の render／serve サブコマンドの引数受理・出力動作が一切変化しないことを保証する
  - _Requirements: 1.1, 1.7_
- [x] 2.2 import サブコマンドの引数受理
  - DSN／ドライバ種別／出力先／タイトル／スキーマ名の各オプションを受け取る
  - 出力先を省略した場合は標準出力へ流し、指定された場合はファイルへ書き出す経路に値を渡す
  - DSN が空または未指定のときは使い方文字列を標準エラーへ出力して非ゼロ終了する
  - _Requirements: 1.2, 1.3, 1.4, 1.6_

- [ ] 3. ドライバ確定と DSN 周辺ロジック
- [x] 3.1 (P) ドライバ自動推定ロジックの実装
  - ドライバ種別の明示指定があれば優先採用し、未指定時は DSN プレフィックスとファイル拡張子から推定する
  - サポート外ドライバ指定時は接続を試行せずエラーで終了する
  - _Requirements: 1.5, 2.4_
- [x] 3.2 (P) MySQL DSN 表記の境界仕様の整理と検証
  - 推定段階では URL スキーム表記を MySQL として識別する一方、接続段階では URL スキーム表記を非推奨と扱い利用者に明示的なエラーを返すか、URL スキーム表記を標準 DSN へ正規化するかのいずれかに方針を統一する
  - 採用方針に応じてエラーメッセージまたは正規化ロジックを実装し、利用者にとって何が起きたかが一目でわかる挙動にする
  - _Requirements: 1.5, 2.2, 11.2_
- [x] 3.3 (P) DSN パスワードマスクの実装
  - PostgreSQL の URL 形式は URL ライブラリによる再構築でパスワード位置を伏字に置換する
  - MySQL の標準 DSN はドライバ提供の構成構造体経由でパスワード位置を伏字に置換する
  - SQLite はパスワード概念を持たない前提でそのまま返すが、ログ出力時には原 DSN を漏らさない方針を維持する
  - _Requirements: 2.5, 10.4_
- [x] 3.4 (P) タイトル既定値解決ロジックの実装
  - タイトル指定があればそれを採用する
  - 未指定時は PostgreSQL／MySQL では DSN から接続先 DB 名を抽出し、SQLite ではファイル名のベース部を採用する
  - DSN パース基盤はパスワードマスクと共有し、操作の散在を避ける
  - _Requirements: 9.5, 9.6_

- [ ] 4. PostgreSQL イントロスペクション
- [x] 4.1 PostgreSQL 接続と読み取り専用トランザクションの確立
  - 接続を確立し疎通確認を行い、失敗時はマスク済み DSN を含むエラーを返す
  - 読み取り専用トランザクションで開始し、処理完了時または異常終了時のいずれでも接続をクローズする
  - _Requirements: 2.1, 2.5, 10.1, 10.2, 10.3_
- [x] 4.2 PostgreSQL のテーブル一覧取得
  - 既定スキーマは public とし、明示指定があれば指定スキーマのみを対象とする
  - システムスキーマ・ビュー・マテリアライズドビュー・外部テーブル・一時テーブルを除外し、実テーブルのみを宣言順序で取得する
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.6_
- [x] 4.3 PostgreSQL のカラム情報取得と型正規化
  - 各テーブルのカラムを宣言順で取得し、物理名・データ型・NOT NULL／UNIQUE／デフォルト値・コメントを集める
  - 自動連番列を識別したら型表記を所定の連番型へ正規化し、デフォルト値出力を抑止する
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5, 4.6, 4.7, 8.1_
- [x] 4.4 PostgreSQL の主キー・外部キー・インデックス取得
  - 主キーの構成カラムと宣言順を取得する
  - 外部キーは複合構成を維持して取得し、参照元カラムの NOT NULL／UNIQUE 性も同時に把握する
  - 主キー起源・UNIQUE 起源を除いた補助インデックスを構成カラム順序付きで取得する
  - _Requirements: 5.1, 5.2, 6.1, 6.2, 6.3, 6.4, 7.1, 7.2, 7.3_

- [ ] 5. MySQL イントロスペクション
- [x] 5.1 MySQL 接続と読み取り専用トランザクションの確立
  - 接続を確立し疎通確認を行い、失敗時はマスク済み DSN を含むエラーを返す
  - 読み取り専用トランザクションで開始し、処理完了時または異常終了時のいずれでも接続をクローズする
  - _Requirements: 2.2, 2.5, 10.1, 10.2, 10.3_
- [x] 5.2 MySQL のテーブル一覧取得
  - 接続先 DB を既定対象とし、明示指定があれば指定スキーマのみを対象とする
  - システムスキーマ群およびビュー・一時テーブルを除外し、実テーブルのみを宣言順序で取得する

- [x] 5.3 MySQL のカラム情報取得と型正規化
  - 各テーブルのカラムを宣言順で取得し、物理名・データ型・NOT NULL／UNIQUE／デフォルト値・コメントを集める
  - 自動増分列を識別したらデフォルト値出力を抑止する
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5, 4.6, 4.7, 8.2_
- [x] 5.4 MySQL の主キー・外部キー・インデックス取得

- [ ] 6. SQLite イントロスペクション
- [x] 6.1 SQLite 接続の確立と書き込み禁止運用
  - 接続を確立し疎通確認を行い、失敗時はエラーを返す
  - 読み取り系のみ発行する運用を守り、処理完了時または異常終了時のいずれでも接続をクローズする
  - _Requirements: 2.3, 10.1, 10.3_
- [x] 6.2 SQLite のテーブル一覧と DDL 原文の取得
  - システムテーブル群と一時テーブルを除外し、実テーブルのみを宣言順序で取得する
  - 後段のコメント抽出に利用するため、各テーブルの作成 DDL 原文も同時に取得する
  - _Requirements: 3.1, 3.2, 3.6_
- [x] 6.3 SQLite のカラム情報取得と型正規化
  - カラム情報をカラム宣言順で取得し、物理名・型・NOT NULL／デフォルト値・主キー判定を集める
  - 単一カラム UNIQUE インデックスからカラム単位の UNIQUE 性を導出する
  - 自動 ROWID 連番列を識別したらデフォルト値出力を抑止する
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5, 4.6, 4.7_
- [x] 6.4 SQLite の主キー・外部キー・インデックス取得
  - 主キー構成と外部キー構成を取得する
  - 主キー起源・UNIQUE 起源を除いた補助インデックスを構成カラム順序付きで取得する
  - _Requirements: 5.1, 5.2, 6.1, 6.2, 6.3, 6.4, 7.1, 7.2, 7.3_
- [x] 6.5 SQLite の DDL 原文からのコメント抽出
  - 作成 DDL 原文を簡易パースし、カラム宣言行末の行コメントを物理名と対応付けて抽出する
  - 抽出不能な場合は空文字列とし、後段の物理名フォールバックに委ねる
  - _Requirements: 8.3_

- [ ] 7. ドメインモデルへの変換
- [x] 7.1 内部 DTO からドメインモデルへの基本マッピング
  - テーブル一覧・カラム一覧・主キー・補助インデックスを既存ドメインモデルへ写像する
  - 取得テーブルが 0 件の場合はエラーを返し、出力を生成しない
  - _Requirements: 3.5, 4.2, 4.3, 4.4, 4.5, 4.6, 5.1, 5.2, 5.3, 7.1, 7.2, 7.3_
- [x] 7.2 (P) 論理名フォールバックとサニタイズの集約
  - 取得済みコメントを論理名として採用し、空のときは物理名を論理名として採用する
  - 論理名にスラッシュが含まれる場合は全角スラッシュへ、改行が含まれる場合は半角スペースへ置換する
  - 適用箇所をテーブル・カラムで共通化し、フォールバック関数を 1 か所に集約する
  - _Requirements: 8.4, 8.5, 8.6, 8.7_
- [x] 7.3 (P) 外部キーのカーディナリティ決定ロジック
  - 参照元カラムが NULL 許容かつ UNIQUE でない場合は多対一を採用する
  - 参照元カラムが NOT NULL かつ UNIQUE でない場合は必須多対一を採用する
  - 参照元カラムが UNIQUE な単一カラム外部キーの場合は一対一を採用する
  - 複合外部キーは構成の先頭カラムにのみ関係を付与し、残余カラムには付与しない
  - _Requirements: 6.1, 6.2, 6.3, 6.4_
- [x] 7.4 スコープ外参照外部キーの除外と警告
  - 取得対象テーブル一覧に存在しない参照先を持つ外部キーは関係を生成しない
  - 当該外部キーについて警告メッセージを標準エラーへ出力し、処理は継続する
  - _Requirements: 6.5_
- [x] 7.5 タイトルとスキーマ全体の組み立て
  - 解決済みタイトルをスキーマのタイトルとして設定する
  - 検証済みのテーブル列を順序保持してスキーマに格納する
  - _Requirements: 3.6, 9.4, 9.5, 9.6_

- [ ] 8. CLI 配線とエラー終了制御
- [x] 8.1 import パイプラインの組み立て
  - 引数解析・ドライバ確定・タイトル解決・接続・スキーマ取得・モデル構築・検証・直列化・出力という直線的な流れを 1 か所で順序通りに配線する
  - 配線関数自体は配線責務に専念させ、各段階の実装を呼び出すに留める
  - _Requirements: 1.1, 1.2, 1.3, 1.4_
- [x] 8.2 検証段階での書き出し抑制
  - スキーマ検証で違反が検出された場合、出力ファイルを生成せず違反内容を標準エラーへ出力して非ゼロ終了する
  - _Requirements: 9.2, 11.3_
- [x] 8.3 出力先ディレクトリの存在検査と書き出し
  - 出力先指定時は親ディレクトリの存在を検査し、存在しない場合は所定のメッセージで非ゼロ終了する
  - 標準出力／ファイルいずれの経路でも書き出し失敗時は非ゼロ終了する
  - _Requirements: 1.3, 1.4, 11.1, 11.2, 11.4_
- [x] 8.4 終了コードとエラーメッセージの統一
  - 成功時はゼロ終了、失敗時は非ゼロ終了に統一する
  - エラー文言の生成箇所では原 DSN を露出させず、必ずマスク済み DSN を経由させる
  - _Requirements: 10.4, 11.1, 11.2_
