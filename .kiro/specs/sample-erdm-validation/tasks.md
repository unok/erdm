# tasks: sample-erdm-validation

設計（`design.md`）と要件（`requirements.md`）に基づく実装タスクリスト。各タスクは設計コンポーネントと要件 ID に紐付く。`(P)` はそのタスクが、同レベル内で並列実行可能であることを示す（前提タスクは完了済みであること）。

- [x] 1. 検証用サンプル `.erdm` を整備する
- [x] 1.1 (P) 最小サンプル DSL を作成する
  - `# Title:` 1 行を含む最小構成（テーブル 2 個、PK 1 個ずつ、単純な FK 1 本）の `.erdm` を `doc/sample/` 配下に配置する
  - パーサが受理可能であることをローカルで `--format=elk` 経由で確認する
  - _Requirements: 1.1, 1.3, 7.1_
- [x] 1.2 (P) 構文網羅サンプル DSL を作成する
  - `# Title:` を 1 行目に配置する
  - `+name` 形式の主キー列を持つテーブルを最低 3 個含める
  - 列属性 4 種（NOT NULL / UNIQUE / DEFAULT / ERD 非表示）をそれぞれ最低 1 回ずつ使用し、同一テーブルに集約しない
  - 異なるカーディナリティ表記を 3 種以上、それぞれ最低 1 回ずつ含める
  - 複合インデックス（複数列指定）を 1 個以上含める
  - ユニーク・インデックスを 1 個以上含める
  - 複数グループ宣言（`@groups[...]`）を 1 個以上含める
  - 列定義直後の列単位コメントを 1 個以上含める
  - 論理名なし列を 1 個以上含める
  - 既存サンプル群の DSL は変更せず、新規ファイルのみ追加する
  - 完成後、パーサが受理することを `--format=elk` で確認する
  - _Requirements: 1.2, 1.4, 1.5, 1.6, 1.7, 1.8, 1.9, 1.10, 1.11, 1.12, 1.13, 7.1_

- [x] 2. 検証スクリプトの基盤ユーティリティを整備する
- [x] 2.1 スクリプトのエントリポイントと前提条件早期検出を実装する
  - 安全な bash オプション（未定義変数禁止・エラー伝播・パイプ失敗検出）を有効化する
  - `dot` / `psql` / `sqlite3` / `jq` / `curl` の有無を検出して内部状態に格納する
  - SPA 同梱物（フロントエンドのビルド成果物のエントリ HTML）の存在を検出する
  - 必須前提条件（HTTP クライアントの存在、SPA 同梱物の存在、入力サンプル 2 種の存在）を満たさない場合はスクリプト冒頭で fatal とし、利用者に解決手順（フロントエンドのビルドコマンド実行）を促すメッセージを標準エラーへ出力して非ゼロ終了する
  - PATH 上に `erdm` バイナリが存在することを前提条件として検出し、不在時は fatal で `make build` 等を促す
  - _Requirements: 6.1, 6.2_
- [x] 2.2 一時ディレクトリ管理と終了時クリーンアップを実装する
  - 専用の一時作業ディレクトリを生成し、スクリプト終了時（正常終了・エラー・割り込みのいずれでも）に確実に削除する
  - 起動中のサーバプロセスがあれば併せて停止する
  - クリーンアップは多重起動・既停止プロセス・既削除ディレクトリのいずれでも安全に動作させる
  - _Requirements: 6.7_
- [x] 2.3 (P) 空きポート探索ユーティリティを実装する
  - 既定のポート起点から所定範囲で空きポートを順次走査し、最初に発見した空きポート番号を返す
  - 走査は外部ツール非依存の手段（bash 標準の TCP 接続試行）で行い python3 等を要求しない
  - 範囲内に空きが見つからない場合は明示エラーで非ゼロ終了する
  - _Requirements: 4.1, 6.5_
- [x] 2.4 (P) アサート集約ヘルパを実装する
  - 完全一致・部分文字列含有・ファイル存在・終了コード一致・HTTP ステータス一致の 5 種比較を提供する
  - 失敗時は標準エラーへ「期待値・実測値・対象要件 ID」を含む診断行を出力し、失敗カウントを内部状態でインクリメントする
  - 失敗時にスクリプト全体を即停止せず、後続検証を継続できる挙動とする
  - スクリプト終了時に失敗カウントを参照し、1 件でも失敗があれば非ゼロ終了する
  - _Requirements: 6.6_
- [x] 2.5 (P) スキップ判定ヘルパを実装する

  - serve ブロック全体のスキップ判定（`dot` 不在時、要件文の「のみ」に厳密準拠）を提供する
  - PostgreSQL 構文検証アサート単位のスキップ判定（`psql` 不在時）を提供する
  - SQLite 構文検証アサート単位のスキップ判定（`sqlite3` 不在時）を提供する
  - スキップ時は標準出力へ対象要件 ID を含む明示ログを出力し、当該検証を成功扱いとする
  - _Requirements: 6.8_

- [x] 3. render(dot) 検証ブロックを実装する

  - 5 種出力（DOT / PNG / HTML / PostgreSQL DDL / SQLite DDL）が出力ディレクトリに生成されることを検証する

  - 生成された SQLite DDL を SQLite のメモリ DB に投入し、終了コードが 0 であることを検証する（`sqlite3` 不在時はスキップ）
  - ブロック全体は `dot` 不在時にスキップ判定で迂回する
  - 検証は要件 2.1 → 2.2 → … → 2.9 の順で直列実行する
